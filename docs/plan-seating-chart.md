## 1. 개요

교회 성가대의 매 예배·행사 좌석 배치를 손쉽게 수정하고 공유할 수 있는 단일 페이지 웹 도구.
지휘자/총무가 PC 또는 모바일 브라우저에서 단원을 좌석에 배치하고, 그 결과를 화면으로 보여주거나 PNG 이미지로 저장해 메신저로 공유한다.

### 핵심 가치
- 설치·로그인 없음. 브라우저로 바로 사용
- 단원 명단·배치 상태는 브라우저 localStorage에 자동 저장
- 모바일에서도 손가락 탭만으로 배치 가능

## 3. 아키텍처

- **형식**: 단일 `index.html` (HTML/CSS/JS 한 파일)
- **외부 의존성**: `html2canvas` CDN 1개 (PNG 다운로드용)
- **상태 관리**: in-memory JS 객체. 모든 변경은 dispatch → state 갱신 → localStorage 저장 → 재렌더의 단방향 흐름
- **반응형**: 화면 너비 768px 미만에서는 좌석 영역만 가로 스크롤, 페이지 전체는 스크롤 안 함

```
┌──────────────────────── index.html ────────────────────────┐
│ [상단 툴바] 레이아웃 편집 / PNG 다운로드 / 초기화           │
│                                                              │
│ ┌─ 단원 명단 (상단) ────────────────────────────────────┐ │
│ │ 초성 필터: [전체][ㄱ][ㄴ][ㄷ]...[ㅎ]                   │ │
│ │ + 단원 추가                                            │ │
│ │ ◯ 김지영  ◯ 김철수  ◯ 박OO  ...                       │ │
│ │ 배치 N / 미배치 M / 총 66명                           │ │
│ └───────────────────────────────────────────────────────┘ │
│                                                              │
│ ┌─ 좌석 배치도 (하단) ──────────────────────────────────┐ │
│ │       [Left Side]              [Right Side]            │ │
│ │  (Left는 오른쪽 정렬, Right는 왼쪽 정렬)              │ │
│ └───────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

## 4. 컴포넌트

### 4-1. 상단 툴바
- **[레이아웃 편집]** — 좌·우 각 9개 행의 인원수 수정 모달
- **[PNG 다운로드]** — 좌석 배치도 영역을 이미지로 저장
- **[초기화]** — 모든 배치 해제 (단원 명단은 유지). 확인창 표시

### 4-2. 단원 명단 패널 (상단)
- 헤더: `배치 N / 미배치 M / 총 66명` 카운터
- 초성 필터 바: `[전체][ㄱ][ㄴ][ㄷ][ㄹ][ㅁ][ㅂ][ㅅ][ㅇ][ㅈ][ㅊ][ㅋ][ㅌ][ㅍ][ㅎ]`
- [+ 단원 추가] 버튼 → 단원 입력 모달
- 단원 카드 리스트: 미배치 인원만, 가나다 오름차순
  - 카드: 이름 + (메모 있으면 작은 글씨로 미리보기)
  - 길게 누르기/우클릭 → 수정·삭제 메뉴
  - 드래그 시작점 또는 탭 선택 시작점

### 4-3. 좌석 배치도 (하단)
- 두 그리드(`.left-side`, `.right-side`)가 가운데 통로(spacer)를 사이에 두고 배치
- 각 행은 별도 row 컨테이너
  - Left row: `justify-content: flex-end` (오른쪽 정렬)
  - Right row: `justify-content: flex-start` (왼쪽 정렬)
- 빈 행(인원 0)은 시각적 통로(여백 행)로 렌더
- **모바일 행 폭 통일** (v1.2): 그리드 컬럼을 `max(left) × seat_w + gap` 고정 폭으로 산정 → 좌석 수가 다른 행도 우측 끝(아일 위치) 일직선 정렬
- **자리번호 표시** (v1.2): 각 좌석에 `{행letter}{2자리번호}` (예: A07)
  - 행 letter는 좌석 있는 행에만 A,B,C... 순서대로 부여(빈 행 건너뜀, 매 렌더 재계산)
  - 좌석 번호는 행 내 좌→우 순번 (left side 1..n, right side n+1..n+m)
  - 빈 좌석: 셀 가운데에 큰 워터마크 자리번호
  - 점유 좌석: 좌상단 미니 라벨 (8px PC / 7px 모바일, 파랑톤)
- 좌석 셀:
  - 빈 좌석: 점선 테두리, 클릭/탭 시 단원 선택 모드 진입
  - 배치된 좌석 (v1.2 색상 교체: 안정감 있는 **파랑** `#e9f1ff`):
    - 단원 이름(중앙) + 메모(아래 작은 글씨)
    - **더블클릭/더블탭** → 즉시 배치 해제
    - **길게 누르기 (500ms)** → 단원 수정 모달 (이름/메모 편집·삭제)
  - 좌석 ↔ 좌석 드래그: 두 단원 교환(스왑)
  - 좌석을 명단 영역으로 드래그: 배치 해제 (PC)

### 4-4. 모달
- 단원 추가/수정 모달: 이름(필수) + 메모(선택, 짧은 텍스트)
- 레이아웃 편집 모달: 좌·우 각 9개 행의 인원수(0~10) 숫자 입력. `합계: ?? / 단원 ??명` 실시간 표시

### 4-5. 인터랙션 모드
- `selectedMember` 또는 `selectedSeat` 중 하나만 활성. 선택된 항목은 파란 테두리로 하이라이트
- **선택 시작점** (둘 중 어느 쪽이든 가능):
  - 단원 카드 탭 → `selectedMember`
  - 빈 좌석 탭 → `selectedSeat`
- **선택 후 동작**:
  - `selectedMember` 상태에서 빈 좌석 탭 → 배치 → 해제
  - `selectedSeat` 상태에서 단원 카드 탭 → 배치 → 해제
- **배치된 좌석의 동작** (v1.2 갱신, × 버튼 제거):
  - **더블클릭/더블탭** → 즉시 배치 해제 (선택 모드 무관)
  - **길게 누르기 (≥500ms)** → 단원 수정 모달 (이름·메모 편집/삭제)
  - 단일 클릭/탭: v1에서는 동작 없음 (이동/스왑은 PC 드래그로만 지원)
  - 모바일에서 배치된 단원을 다른 자리로 옮기려면: 더블탭으로 해제 → 카드 탭 → 새 좌석 탭 (2단계)
- **선택 해제**: 빈 곳 탭, ESC 키, 또는 같은 항목 재탭

## 5. 데이터 흐름 & 저장 구조

### 5-1. 흐름

```
[사용자 액션] → [reducer(dispatch)] → [state 갱신] → [localStorage 저장 + 화면 재렌더]
```

### 5-2. state 스키마

```js
state = {
  members: [
    { id: "m_1718...", name: "김지영", memo: "" },
    { id: "m_1719...", name: "박철수", memo: "솔로" },
    // ... 총 66명
  ],
  layout: {
    left:  [6, 6, 6, 0, 6, 6, 0, 7, 7],  // 9개 행, 0은 빈 행
    right: [2, 3, 0, 0, 4, 4, 0, 4, 5]
  },
  assignments: {
    "left-0-0":  "m_1718...",
    "left-0-1":  "m_1719...",
    "right-1-2": "m_1720...",
    // 미배치 좌석은 키 없음
  }
}
```

미배치 단원 계산:
`members.filter(m => !Object.values(assignments).includes(m.id))`

### 5-3. localStorage

- **키**: `seating-chart-v1`
- **값**: `JSON.stringify(state)` 전체 직렬화
- **저장 시점**: 모든 dispatch 마지막 단계
- **로드 시점**: 페이지 로드 직후 1회.
  - 저장된 데이터 있음 → 그대로 사용
  - 저장된 데이터 없음 → **초기 시드 데이터**로 시작 (members는 부록 A의 66명, layout은 위 표 값, assignments는 빈 객체)
- **버전 키 `v1`**: 향후 스키마 변경 시 마이그레이션 여지

### 5-4. 액션 목록

| 액션 | 페이로드 | 효과 |
|---|---|---|
| `ADD_MEMBER` | `{name, memo}` | members에 추가 |
| `UPDATE_MEMBER` | `{id, name, memo}` | 해당 단원 수정 |
| `DELETE_MEMBER` | `{id}` | members에서 제거, assignments에서도 해당 좌석 비우기 |
| `ASSIGN` | `{seatKey, memberId}` | 단원 → 좌석 배치 (해당 단원이 다른 좌석에 있으면 자동 해제) |
| `UNASSIGN` | `{seatKey}` | 좌석 비우기 (단원은 명단으로 복귀) |
| `SWAP` | `{seatKeyA, seatKeyB}` | 두 좌석 단원 교환 |
| `UPDATE_LAYOUT` | `{side, rowCounts}` | 레이아웃 변경, 사라진 좌석의 assignments는 자동 해제 |
| `RESET_ASSIGNMENTS` | — | assignments만 비움, members/layout 유지 |

### 5-5. 한글 초성 추출

```js
function getChosung(name) {
  const CHOSUNG = ['ㄱ','ㄲ','ㄴ','ㄷ','ㄸ','ㄹ','ㅁ','ㅂ','ㅃ',
                   'ㅅ','ㅆ','ㅇ','ㅈ','ㅉ','ㅊ','ㅋ','ㅌ','ㅍ','ㅎ'];
  const code = name.charCodeAt(0) - 0xAC00;
  if (code < 0 || code > 11171) return null;          // 한글 외
  return CHOSUNG[Math.floor(code / 588)];
}
```
- 쌍자음(ㄲ/ㄸ/ㅃ/ㅆ/ㅉ)은 필터에서 평음(ㄱ/ㄷ/ㅂ/ㅅ/ㅈ)으로 묶어 처리

### 5-6. 정렬

```js
members.slice().sort((a, b) => a.name.localeCompare(b.name, 'ko'))
```

## 6. 에러 처리 & 엣지 케이스

### 6-1. localStorage 관련

| 상황 | 처리 |
|---|---|
| localStorage 사용 불가(시크릿 모드 등) | 상단 경고 배너: "저장이 비활성화되어 새로고침 시 데이터가 사라집니다". 동작은 in-memory로 계속 |
| 저장 데이터 JSON 파싱 실패 | 백업 키(`seating-chart-v1.broken`)로 원본 이동 후 빈 state로 시작. 사용자에게 알림 |
| `QuotaExceededError` | 거의 발생 안 함(66명·텍스트) — 발생 시 alert |

### 6-2. 단원 관련

| 상황 | 처리 |
|---|---|
| 빈 이름 추가 시도 | 모달 저장 버튼 비활성화 + 안내 |
| 동명이인 | 허용 (id로 구분). 명단에 동일 이름이 나란히 표시 |
| 배치된 단원 삭제 | 확인 모달: "OOO 단원을 삭제하면 배치된 좌석도 비워집니다" |

### 6-3. 좌석/배치 관련

| 상황 | 처리 |
|---|---|
| 이미 배치된 좌석에 드롭 | 두 단원 자동 스왑 |
| 좌석에 있는 단원을 다른 좌석으로 드래그 | 원래 자리는 자동 빈 좌석 |
| 좌석을 명단 영역으로 드래그 | 배치 해제 |
| 선택 모드에서 이미 선택된 항목 재탭 | 선택 해제 |
| ESC 또는 빈 영역 탭 | 선택 해제 |

### 6-4. 레이아웃 변경 관련

| 상황 | 처리 |
|---|---|
| 행 인원 줄여서 배치된 좌석이 사라짐 | 저장 시 사전 경고: "변경 시 OOO, XXX 단원이 명단으로 돌아갑니다" |
| 좌·우 합계 ≠ 단원 수 | 차단하지 않음. 모달 하단에 정보성 문구: `좌석 67 / 단원 66 → 미배치 가능` |
| 모든 행이 0 | 저장 차단 + 안내 |

### 6-5. PNG 다운로드 관련

| 상황 | 처리 |
|---|---|
| `html2canvas` 로드 실패 | alert: "이미지 라이브러리를 불러올 수 없습니다. 인터넷 연결을 확인하세요" |
| 매우 좁은 모바일 화면 | 다운로드 직전 좌석 영역을 임시로 데스크탑 폭으로 렌더 후 캡처, 끝나면 원복 |
| 파일명 | `seating-chart-YYYYMMDD-HHmm.png` |

### 6-6. 반응형 / 입력 디바이스

| 상황 | 처리 |
|---|---|
| 모바일 좁은 화면 | 좌석 영역만 가로 스크롤. 통로 위치 시각적 유지 |
| 터치 디바이스 | HTML5 DnD는 터치 미지원 → **탭 선택이 주 입력**. 드래그는 PC 보조 |
| 더블 탭/실수 탭 | 단일 탭 = 선택, 다음 탭 = 배치/해제로 단순 처리 |

## 7. 테스트 전략

### 7-1. 철학

단일 HTML·의존성 최소·단일 사용자 → 무거운 테스트 인프라 대신 **순수 함수 인라인 자가테스트 + 수동 체크리스트**.

### 7-2. 자가테스트 (URL `?test=1`)

순수 함수 대상:
1. `getChosung(name)` — "김지영"→"ㄱ", "끔"→"ㄱ", "Eric"→null
2. `sortMembersByName(members)` — 가나다 정렬
3. `filterByChosung(members, chosung)` — null이면 전체
4. `getUnassignedMembers(members, assignments)` — 미배치 계산
5. `seatKey(side, row, col)` / `parseSeatKey(key)` — 좌석 키 변환
6. `getTotalSeats(layout)` — 좌석 총합 (검증값: 66)
7. `validateLayout(layout, memberCount)` — 합계 비교 결과

## 9. v1.2 변경사항 (2026-06-27)

### UX 개선
- **자리번호 표시**: 모든 좌석에 `{행letter}{2자리번호}` 표기 (예: A07)
  - 행 letter는 좌석 있는 행에만 A부터 순서대로 — 빈 행은 letter 미할당 → 레이아웃 편집으로 빈 행에 좌석이 생기면 자동 재정렬
  - 빈 좌석: 가운데 큰 워터마크
  - 점유 좌석: 좌상단 미니 라벨 (이름과 시야 분리)
- **× 버튼 제거** → 점유 좌석 더블클릭/더블탭으로 배치 해제
- **길게 누르기 (500ms)** → 단원 수정 모달 (이름·메모 편집/삭제)
- **메모 폰트 분리**: 이름 12px / 메모 9px (이름 > 메모로 시각적 우선순위)
- **모바일 행 정렬 일치**: 좌석 수 다른 행끼리 우측 끝(아일 위치)이 일직선
- **색상 컨벤션 교체**:
  - 명단 카드(미배치): **노랑** `#fffae6` / border `#d4b942` — "조치 대기" 표시
  - 점유 좌석(배치 완료): **파랑** `#e9f1ff` / border `#b3cdf6` — "안정" 표시
- **모바일 더블탭 줌 지연 제거**: `touch-action: manipulation`
